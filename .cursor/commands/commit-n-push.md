Faça commit e push das alterações deste escopo.

Passos:
1. Analise o contexto da tarefa para identificar os arquivos relevantes
2. Execute `git status` para revisar as alterações
3. Adicione apenas os arquivos necessários com `git add`
4. Gere uma mensagem de commit clara no formato:
   `[tipo]: descricao`

Tipos permitidos:
- feat (nova funcionalidade)
- fix (correção de bug)
- refactor (refatoração)
- chore (infra/ajustes técnicos)
- docs (documentação)

Regras:
- Não incluir arquivos não relacionados
- Não fazer commit vazio
- Não modificar arquivos adicionais
- Mensagem em português, objetiva

Finalize com:
`git push`
